# 小红书博主总结 Skill

## 适用场景

- 用户给出小红书博主主页链接，要求总结其**所有内容/前N条笔记**
- 用户要求"总结这个博主的笔记"、"帮我看看这个博主发了什么"
- 博主内容混合**视频笔记 + 图文笔记**，两者都需要总结
- 需要逐条获取每条笔记的详细内容和核心观点

## 前置条件

1. **OpenCLI 已安装并可用**：`opencli` 命令生效，Chrome 浏览器扩展已加载
2. **网络通畅**：代理设置正确（通常为 `http://127.0.0.1:7890`）
3. 部分博主内容可能需要登录态，如遇权限限制需提醒用户

## 工作流

### Step 1: 提取博主 UID

从小红书博主主页链接中提取 UID（十六进制数字串）。

示例：
```
链接: https://www.xiaohongshu.com/user/profile/578bb69e82ec3963601d61a4
UID: 578bb69e82ec3963601d61a4
```

### Step 2: 获取博主笔记列表

```bash
opencli xiaohongshu user <UID> -f yaml --window background --limit <数量> 2>&1 | head <行数>
```

- `--limit` 控制获取的笔记数量（建议 30-50 条）
- 从输出提取每条笔记的：标题、类型（video/image）、点赞数、笔记 ID
- 按时间倒序排列（最新在前）

### Step 3: 筛选目标笔记

按用户需求截取前 N 条（如"总结前20条"则取前20条）。

### Step 4: 逐条获取详细内容

**4.1 视频笔记**

对每条 video 笔记，用 `opencli xiaohongshu ask` 获取 AI 视频内容总结：

```bash
opencli xiaohongshu ask "博主名 笔记标题关键词 视频内容详细内容" --source-limit 10 2>&1 | head 400
```

示例：
```bash
opencli xiaohongshu ask "文翰的城市更新实验室 阿云朵仓 4000方1个亿 视频详细内容" --source-limit 10 2>&1 | head 400
```

**4.2 图文笔记**

对每条 image 笔记，用同样方法获取详细内容：

```bash
opencli xiaohongshu ask "博主名 笔记标题关键词 图文内容详细内容" --source-limit 10 2>&1 | head 400
```

注意：
- 关键词组合：博主名 + 笔记标题核心词 + "视频内容" 或 "图文内容" 或 "详细内容"
- `--source-limit` 设为 5-10（内容多时用 10）
- 如果某条搜索结果为空，跳过并继续下一条

### Step 5: 整理汇总生成报告

将每条笔记的详细信息整合为一份完整报告：

| 字段 | 说明 |
|------|------|
| 笔记序号 | 按时间倒序编号 |
| 标题 | 笔记完整标题 |
| 类型 | video（视频）/ image（图文） |
| 互动数据 | 点赞数、收藏数 |
| 标签/关键词 | 从笔记元数据中提取 |
| 内容摘要 | 从 opencli xiaohongshu ask 获取的详细解读 |
| 核心观点 | 总结该笔记的核心论点/发现 |
| 涉及人物/项目 | 视频中提到的具体人名、地名、商业项目 |

### Step 6: 输出报告

将完整报告写入文件：
```
outputs/<博主名>_前N条笔记深度总结.md
```

## 常见问题处理

| 问题 | 解决方案 |
|------|---------|
| `opencli` 命令不存在 | 安装 OpenCLI：`npm install -g @jackwener/opencli`，加载 Chrome 扩展 |
| 视频/图文内容摘要为空 | 调整关键词或用 `opencli xiaohongshu note <笔记ID>` 直接获取 |
| 代理超时 | 检查代理设置，尝试更换端口 |
| 笔记数量为 0 | 博主可能为新号或设置了隐私权限 |
