# 小红书探测器 Skill

## 适用场景

- 用户给出小红书博主主页链接，要求总结其**所有内容/前N条笔记**
- 用户要求"总结这个博主的笔记"、"帮我看看这个博主发了什么"
- 博主内容混合**视频笔记 + 图文笔记**，两者都需要总结
- 需要逐条获取每条笔记的详细内容和核心观点
- 发现优质博主后预筛选，判断是否值得深入关注

## 前置条件

1. **OpenCLI 已安装并可用**：`opencli` 命令生效，Chrome 浏览器扩展已加载
2. **网络通畅**：代理设置正确（通常为 `http://127.0.0.1:7890`）
3. 部分博主内容可能需要登录态，如遇权限限制需提醒用户
4. 建议用户关注的博主领域与笔记内容领域匹配，提高搜索结果质量

## 工作流

### Step 1: 提取博主 UID

从小红书博主主页链接中提取 UID（十六进制数字串）。

示例：
```
链接: https://www.xiaohongshu.com/user/profile/578bb69e82ec3963601d61a4
UID: 578bb69e82ec3963601d61a4
```

### Step 2: 获取博主笔记列表

```bash
opencli xiaohongshu user <UID> -f yaml --window background --limit <数量> 2>&1 | head <行数>
```

- `--limit` 控制获取的笔记数量（建议 30-50 条）
- 从输出提取每条笔记的：标题、类型（video/image）、点赞数、收藏数、笔记 ID、标签
- 按时间倒序排列（最新在前）
- **跳过置顶文章**（如果用户要求）

### Step 3: 筛选目标笔记

按用户需求截取前 N 条（如"总结前20条"则取前20条）。

### Step 4: 逐条获取详细内容（含进度追踪+容错重试+多关键词策略+严格单条搜索）

**4.1 进度追踪表初始化**

在开始搜索前，创建进度追踪表：

| 笔记序号 | 标题 | 类型 | 搜索状态 | 关键词方案 | 结果长度 | 备注 |
|---------|------|------|---------|-----------|---------|------|
| 1 | ... | video | pending | - | - | - |

**4.2 多关键词搜索策略**

对每条笔记，按以下优先级尝试关键词方案：

**方案A（主关键词）** — 最长且最具描述性的标题核心词：
```bash
opencli xiaohongshu ask "博主名 [笔记标题前6个字] [关键词1] [关键词2] 详细内容" --source-limit 10 2>&1 | head 400
```

**方案B（备选关键词）** — 如果方案A结果为空或太短（<100字），换用备选关键词：
```bash
opencli xiaohongshu ask "博主名 [笔记标签/话题词] [相关内容] 详细内容" --source-limit 10 2>&1 | head 400
```

**方案C（兜底搜索）** — 如果方案B仍为空，尝试更广泛的搜索：
```bash
opencli xiaohongshu ask "博主名 [笔记ID] [标题全文] 详细内容" --source-limit 10 2>&1 | head 400
```

**注意**：
- `--source-limit` 设为 5-10（内容多时用 10）
- 每次搜索后检查结果长度，>100字视为成功
- 如果连续3次搜索都为空，标记为"无法获取"并记录原因

**4.3 严格单条搜索（保证质量）**

采用严格单条搜索策略，每条笔记单独调用一次 `opencli xiaohongshu ask`，确保每条都有独立、精准的分析。

```bash
opencli xiaohongshu ask "博主名 笔记标题核心词 详细内容" --source-limit 10 2>&1 | head 400
```

**优势**：
- 每条笔记分析独立，不会互相混淆
- 结果更精准，细节更丰富
- 进度可控，哪条失败一目了然

**劣势**：
- API 调用次数较多（20条=20次调用）
- 整体耗时较长

**注意**：
- 如果笔记数量超过 30 条，建议分批处理（每批 10-15 条）
- 对于高互动笔记（TOP 5），即使搜索成功也建议二次验证，确保核心观点准确

**4.4 容错重试机制**

在进度追踪表中实时更新状态：

- **搜索成功**：标记 `success`，记录结果长度
- **搜索失败（重试后）**：标记 `fail_retry`，记录原因
- **完全无法获取**：标记 `fail_permanent`，在报告中注明"该笔记因 XX 原因未能获取详细内容"

**整体成功率检查**：
- 成功率 ≥ 70%：正常生成报告，并在报告末尾标注"详细解读覆盖率 XX%"
- 成功率 50%-70%：警告用户"部分笔记内容不完整，报告可能存在信息缺失"
- 成功率 < 50%：中断流程，提示用户"当前博主内容获取率过低，请检查网络或代理设置后重试"

### Step 5: 整理汇总生成报告（含时间线+关联分析+方法论）

将每条笔记的详细信息整合为一份完整报告：

| 字段 | 说明 |
|------|------|
| 笔记序号 | 按时间倒序编号 |
| 标题 | 笔记完整标题 |
| 类型 | video（视频）/ image（图文） |
| 互动数据 | 点赞数、收藏数 |
| 标签/关键词 | 从笔记元数据中提取 |
| 内容摘要 | 从 opencli xiaohongshu ask 获取的详细解读 |
| 核心观点 | 总结该笔记的核心论点/发现 |
| 涉及人物/项目 | 视频中提到的具体人名、地名、商业项目 |

### Step 6: 输出报告（增强版模板）

将完整报告写入文件：
```
outputs/<博主名>_前N条笔记深度总结.md
```

## 备用方案：仅提供单条笔记 explore 链接时

如果用户只提供了单条笔记链接（如 `https://www.xiaohongshu.com/explore/xxxxx`），而**不是博主主页链接**，按以下步骤操作：

1. **从笔记链接提取博主名**：使用 `opencli xiaohongshu ask "小红书 <笔记ID或关键词> 博主笔记内容"` 进行 AI 搜索，从搜索结果中提取博主名和相关笔记信息
2. **获取博主其他笔记**：基于第一步得到的博主名，用 `opencli xiaohongshu ask "博主名 关键词1 关键词2 ..."` 多角度搜索，收集该博主的其他笔记标题和链接
3. **逐条获取详细内容**：对每条笔记用关键词组合（博主名 + 笔记标题核心词 + "详细内容"）搜索，获取 AI 详细解读
4. **跳过置顶文章**：如果用户要求跳过置顶文章，在整理笔记时排除置顶的条目

注意：这种情况下无法精确获取点赞数和收藏数等元数据，报告中应标注数据来源和推测依据。
| 批量搜索结果混乱 | 退回单条搜索，逐条获取 |
| 成功率 < 50% | 检查网络连接，确认代理设置，重试 3 次后仍失败则告知用户 |

## 输出报告文件命名规范

```
outputs/<博主名>_前N条笔记深度总结.md
```

示例：
- `outputs/文翰的城市更新实验室_前20条笔记深度总结.md`
- `outputs/建筑师黄伟_前9条笔记深度总结.md`
- `outputs/张咋啦_前20条笔记内容深度总结.md`
